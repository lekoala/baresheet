## 2024-05-15 - [Optimization: Optimized XML Escaping]
**Learning:** `escapeXml` and `escapeXmlAttr` functions were called thousands of times with strings that did not require escaping, spending a large portion of execution time doing redundant checks and allocations. Using `strpbrk` to check if string replacement operations (`htmlspecialchars` or `str_replace`) are even necessary provides a measurable performance boost. Because `strpbrk` is a very fast native C function, using it to skip the more expensive PHP string replacements speeds up execution significantly for common case strings. `str_replace` is generally faster than `htmlspecialchars` and can be used to manually replace standard XML entities where `ENT_XML1 | ENT_COMPAT` handling is not strictly required.
**Action:** When working with string replacement/escaping functions that are repeatedly called in hot loops, check if replacing is necessary first using fast native string search functions like `strpbrk` or `strpos` if the probability of the string needing replacement is low.
